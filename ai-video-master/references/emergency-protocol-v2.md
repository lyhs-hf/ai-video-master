## 触发方式

### 方式1：自然语言触发（推荐）

用户可以说任何自然语言，系统自动识别关键词：

| 用户说 | 系统识别 | 触发 |
|--------|---------|------|
| "这个镜头AI生成失败了" | 失败+镜头 | 肢体/物理急救 |
| "画面崩了" | 崩了 | 肢体急救 |
| "人物变形了" | 变形+人 | 肢体急救 |
| "动作不自然" | 动作+不自然 | 肢体急救 |
| "咖啡洒不出来" | 物理+液体 | 物理急救 |
| "文件掉落的画面不行" | 掉落+文件 | 物理急救 |
| "字幕乱码了" | 乱码+字幕 | 文字急救 |
| "人脸不对了" | 人脸+不对 | 角色急救 |
| "角色变年轻了" | 角色+变化 | 角色急救 |
| "场景颜色不对" | 颜色+场景 | 场景急救 |

**系统响应模板：**

```
检测到问题：[识别结果]
建议急救方案：[对应的急救预案]

是否需要我重新生成？
- 回复"是" → 重新生成
- 回复"不用了" → 保持当前版本
```

## 急救方案速查

### 8.1 肢体融合急救

**问题：** 两人互动画面崩坏（手部畸形、肢体粘连）

**快速方案：**

```
❌ 原方案：两人握手对话（中景）
✅ 急救A：角色A主观视角特写（只看B的上半身）
✅ 急救B：角色B反应特写（只看A说话时的表情）
✅ 急救C：两人都看向镜头外（第三方视角）
```

---

### 8.2 物理规律急救

**问题：** 物品掉落/液体流动/爆炸效果崩坏

**快速方案：**

```
❌ 原方案：咖啡洒在桌上的过程（动态）
✅ 急救A：直接拍桌上已经有一滩咖啡渍（结果）

❌ 原方案：文件从手中滑落（动态）
✅ 急救B：直接拍文件在地上（结果）

❌ 原方案：雨滴落在窗户上（动态）
✅ 急救C：窗户上已经有水痕（结果）
```

---

### 8.3 文字乱码急救

**问题：** 字幕/文件标题/屏幕画面中文乱码

**快速方案：**

```
❌ 原方案：电脑屏幕上显示"财务报表"
✅ 急救A：只拍屏幕发光，关闭屏幕或模糊处理
✅ 急救B：换成纯图形界面（图表、代码）
✅ 急救C：改用图片传递信息而非文字
```

---

### 8.4 角色漂移急救

**问题：** 同一角色在不同镜头间外观差异大

**快速方案：**

```
急救A：强制使用Step 0的锚点图作为垫图
急救B：减少该角色的镜头数量
急救C：将该角色改为背影/侧面/局部特写（减少正脸）
```

---

### 8.5 场景风格急救

**问题：** 不同场景色调/风格差异大

**快速方案：**

```
急救A：统一后期调色（向主场景靠拢）
急救B：减少场景切换（合并相近场景）
急救C：使用黑场/白场转场（掩盖风格差异）
```

## 自检流程

当用户报告问题时，Agent必须：

```
1. 识别问题类型
   ├── 肢体/变形/畸形 → 8.1肢体急救
   ├── 物理/液/掉落 → 8.2物理急救
   ├── 文字/乱码/看不清 → 8.3文字急救
   ├── 角色/人不对/变脸 → 8.4角色急救
   └── 场景/颜色/风格 → 8.5场景急救

2. 提供急救方案（3选1）

3. 询问用户：
   "我已经准备了3个替代方案，是否需要我重新生成？"

4. 根据用户回复执行或结束
```

## 常见问题速查

| 问题 | 可能原因 | 首选急救 |
|------|---------|---------|
| 手部畸形 | 复杂交互、视角问题 | 8.1-A视角特写 |
| 脸部变形 | 提示词不精确 | 8.4-C局部特写 |
| 动作僵硬 | 模型能力限制 | 8.1-B反应镜头 |
| 物体穿透 | 物理模拟问题 | 8.2-A静态结果 |
| 液体乱飞 | 粒子模拟崩坏 | 8.2-C结果展示 |
| 文字乱码 | 中文字符问题 | 8.3-A模糊处理 |
| 角色不一致 | 资产未固化 | 8.4-A锚点垫图 |
| 色调不一 | 提示词差异 | 8.5-A统一调色 |
