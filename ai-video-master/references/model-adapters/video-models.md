# 视频模型适配指南

> 适用场景：文生视频、图生视频、短视频片段生成

---

## 1. 可灵 Kling

### 🎯 定位
快手可灵大模型，**动作连贯性最强**，中文友好，优先图生视频模式。

### 📝 提示词模板

**图生视频（推荐）**
```
[上传图片]

运镜描述：[镜头运动方式]
主体动作：[人物/物体动作描述]
氛围：[光线/氛围描述]
时长：5秒
```

**文生视频**
```
主体：[描述画面主体]
动作：[具体动作描述]
镜头：[镜头运动]
光线：[光线描述]
风格：[风格]
时长：5秒，流畅运动
```

### ⚙️ 参数设置建议

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 模式 | `图生视频` | 比文生视频效果更稳定 |
| 比例 | `9:16` | 竖版短视频 |
| 时长 | `5秒` | 目前最长5秒 |
| 运动笔刷 | 可添加 | 指定局部运动区域 |

**提示词格式**：
- **中文优先**：可灵中文理解最强
- **动作明确**：动词优先，描述具体动作
- **镜头指定**：`固定镜头`、`推进`、`拉远`、`摇镜`
- **时长控制**：5秒内，动作简单明确

**运镜参数建议**：
| 运镜类型 | 提示词示例 |
|----------|------------|
| 固定镜头 | `镜头固定，正面中景` |
| 推进 | `镜头缓慢推进，特写面部` |
| 拉远 | `镜头逐渐拉远，展示全景` |
| 横摇 | `镜头缓慢右摇` |
| 竖摇 | `镜头从脚向上摇` |

### ❌ 常见失败模式 & 解决方案

| 失败现象 | 原因 | 解决方案 |
|----------|------|----------|
| 人物动作僵硬 | 图生视频时参考图质量差 | 使用 Midjourney/即梦 生成的高清图作为输入 |
| 画面闪烁/跳变 | 动作幅度太大 | 减小动作幅度，或分段生成后拼接 |
| 手指/脚部畸形 | 模型对细节处理弱 | 避免手脚特写，或后期修复 |
| 背景扭曲 | 复杂背景干扰 | 简化背景，或使用纯色背景图 |
| 比例变形 | 竖版构图不熟悉 | 明确指定 `9:16竖版`，并确保主体在画面中央 |

### ✅ 优缺点分析

**优点**：
- **动作连贯性最强**（国产最强）
- 中文理解好，提示词直接有效
- 图生视频效果稳定
- 运镜功能丰富
- 每月有免费额度

**缺点**：
- 生成速度较慢（排队时间长）
- 最长只有5秒
- 复杂场景理解有限
- 有时会过度平滑

## 2. 海螺 Hailuo

### 🎯 定位
MiniMax 出品，**动作自然、情感表达好**，擅长人物微表情和细腻动作。

### 📝 提示词模板

**人物情感视频**
```
[上传图片]

情感表达：[描述情感：微笑/悲伤/惊讶]
动作细节：[身体语言描述]
环境互动：[与环境物品的互动]
光线氛围：[光线描述]
时长：6秒
```

**场景视频**
```
主体：[场景/物体描述]
动态元素：[正在发生的变化]
氛围：[整体感觉]
时长：6秒
```

### ⚙️ 参数设置建议

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 模式 | `图生视频` | 情感表达更稳定 |
| 比例 | `9:16` | 竖版 |
| 时长 | `6秒` | 最长6秒 |
| 负面提示词 | 可添加 | 排除不需要的元素 |

**提示词格式**：
- **情感词优先**：`微笑`、`眼神闪烁`、`轻轻皱眉`
- **微动作描述**：`手指轻敲桌面`、`头发随风飘动`
- **氛围词**：`温馨`、`紧张`、`宁静`
- **中文友好**：直接中文描述即可

**情感类提示词库**：
| 情感 | 提示词示例 |
|------|------------|
| 开心 | `嘴角上扬，眼睛弯成月牙，轻轻笑出声` |
| 悲伤 | `眼眶泛红，嘴角下垂，肩膀微微颤抖` |
| 惊讶 | `眼睛睁大，嘴巴微张，往后退了一步` |
| 沉思 | `眉头微皱，目光凝视前方，手指轻触下巴` |
| 浪漫 | `深情对视，手指轻触脸颊，微笑` |

### ❌ 常见失败模式 & 解决方案

| 失败现象 | 原因 | 解决方案 |
|----------|------|----------|
| 情感表达夸张/假 | 提示词太笼统 | 添加微动作细节：`嘴角微微上扬` vs `开心地笑` |
| 动作幅度小 | 提示词动作不够明确 | 添加 `幅度`、`力度` 描述 |
| 多人场景混乱 | 模型处理多人能力有限 | 优先单人场景，或明确主次关系 |
| 服装飘动不自然 | 布料物理模拟难 | 避免大风/飘裙场景，选择小幅度动作 |
| 眼神飘忽 | 眼球运动难控制 | 避免特写眼神，描述 `目光稳定看向某处` |

### ✅ 优缺点分析

**优点**：
- **情感表达最好**，微表情丰富
- 动作自然流畅
- 中文理解优秀
- 最长6秒（比特可灵长）
- MiniMax 生态集成

**缺点**：
- 动作幅度整体偏小
- 复杂场景有时不稳定
- 运镜功能比特可灵少
- 峰值时排队时间长

## 3. Runway

### 🎯 定位
欧美顶级 AI 视频模型，**电影感强、运镜参数丰富**，适合国际化内容和高品质短片。

### 📝 提示词模板

**电影感镜头**
```
[上传图片]

Camera: [镜头类型：PRIME/DRONE/PAN/TILT]
Movement: [运动方式：STATIC/PUSH IN/PULL OUT/TRACKING]
Frame: [景别：EXTREME CLOSE UP/CLOSE UP/MEDIUM/WIDE]
Mood: [氛围：mysterious/tense/serene]
Duration: 5 seconds
```

**创意运镜**
```
[描述] with cinematic camera movement, [具体运镜描述], film grain, color graded, 24fps
```

### ⚙️ 参数设置建议

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 比例 | `9:16` | 竖版 |
| 时长 | `5秒`（Pro） | 标准版5秒 |
| 运动强度 | `Auto/High` | 根据需求调整 |
| 种子 | 可固定 | 保证风格一致性 |

**运镜参数系统**（Runway 特色）：
| 运镜类型 | Gen-3 参数 | 效果 |
|----------|------------|------|
| 固定镜头 | `Camera: STATIC` | 稳如三脚架 |
| 推进 | `Movement: PUSH IN` | 聚焦主体 |
| 拉远 | `Movement: PULL OUT` | 展示环境 |
| 环绕 | `Movement: ORBIT` | 360度环绕 |
| 航拍 | `Camera: DRONE` | 俯视全景 |

**英文提示词优先**：
- Runway 对英文理解更精准
- 使用电影术语：`dolly shot`, `rack focus`, `荷兰角`
- 添加风格词：`film grain`, `anamorphic`, `color graded`

### ❌ 常见失败模式 & 解决方案

| 失败现象 | 原因 | 解决方案 |
|----------|------|----------|
| 过度模糊/噪点 | 运动强度过高 | 降低运动强度 |
| 风格太"AI味" | 默认参数问题 | 添加 `raw`, `cinematic`, `film grain` |
| 人物身份不稳定 | 长视频多人脸漂移 | 分段生成，每段重新定义主体 |
| 运镜不自然 | 运动路径描述不清 | 使用预设运镜模板而非自定义 |
| 饱和度失真 | HDR处理问题 | 添加 `natural lighting`, `muted colors` |

### ✅ 优缺点分析

**优点**：
- **电影感最强**，调色专业
- 运镜参数最丰富
- 生成质量高
- API 支持好（适合自动化）
- 持续迭代更新快

**缺点**：
- **需要英文提示词**（中文理解弱）
- 价格较高
- 对亚洲面孔理解一般
- 国内访问不稳定
- 排队时间长

## 4. Pika

### 🎯 定位
快速生成的 AI 视频工具，**上手简单、速度快**，适合简单场景和快速原型。

### 📝 提示词模板

**简单场景**
```
[上传图片]

Action: [主要动作描述]
Setting: [环境描述]
Duration: 3-4 seconds
```

**动画风格**
```
[描述], animation style, smooth motion, 9:16 vertical
```

### ⚙️ 参数设置建议

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 比例 | `9:16` | 竖版 |
| 时长 | `3-4秒` | 速度最快 |
| 运动强度 | `Mid` | 平衡质量和稳定性 |
| 风格 | 可选 | 动画/写实 |

**提示词格式**：
- **简洁明了**：Pika 擅长简单指令
- **避免复杂场景**：单一主体+简单动作
- **动画效果**：添加 `animation`, `cartoon style` 等

**适用场景类型**：
| 场景类型 | 提示词示例 |
|----------|------------|
| 产品展示 | `产品360度旋转展示，纯白背景` |
| 简单动画 | `角色走路循环，2D动画风格` |
| 表情动画 | `头像微笑，眼睛眨动` |
| 场景转换 | `窗外风景从白天变成夜晚` |

### ❌ 常见失败模式 & 解决方案

| 失败现象 | 原因 | 解决方案 |
|----------|------|----------|
| 动作重复/循环 | 时长太短 | 生成3-4秒，动作自然过渡 |
| 背景糊成一片 | 运动强度过高 | 降低运动强度 |
| 主体变形 | 复杂动作超出能力 | 简化动作，聚焦局部 |
| 生成失败 | 服务器问题 | 稍后重试，或降低参数 |
| 比例失调 | 竖版不熟悉 | 明确指定 9:16 |

### ✅ 优缺点分析

**优点**：
- **速度最快**，几乎不用排队
- 上手最简单
- 免费额度多
- 适合快速原型
- 界面友好

**缺点**：
- 质量一般（电影感不足）
- 动作幅度有限
- 复杂场景处理差
- 运镜功能少
- 细节处理粗糙

## 🎬 短视频制作模型组合实战

### 剧情类短视频（情感向）

```
分镜1 → Midjourney 生成封面图
分镜1 → 海螺 Hailuo（图生视频）→ 情感微表情
分镜2 → Midjourney 生成封面图
分镜2 → 可灵 Kling（图生视频）→ 动作连贯
分镜3 → 即梦 生成封面图
分镜3 → 海螺 Hailuo（图生视频）→ 情感表达
...
```

### 快节奏/电商类短视频

```
封面图 → Midjourney/即梦（电影感）
视频片段 → Pika（快速批量生成）
运镜增强 → Runway（专业运镜）
```

### 国际化内容

```
封面图 → Midjourney（全球审美）
视频 → Runway（欧美电影感）
对口型 → HeyGen（多语言支持）
```

## 🔧 分段视频拼接 SOP

当需要生成长视频时（>10秒）：

1. **规划分镜**：将脚本拆分为 5-6 秒的片段
2. **逐段生成**：每个片段单独生成
3. **风格统一**：固定关键提示词（如光线、色调）
4. **使用剪映拼接**：添加转场、调色统一
5. **BGM 同步**：根据视频节奏选择合适 BGM
